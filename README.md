# ハーブブレンド管理アプリ Herb Recipe Lab

# **1. サービス概要**

このサービスは、ハーブティーのブレンドレシピを記録・整理・共有できるアプリです。
ハーブの配合、味のメモ、期待する効果などをまとめて管理できます。
ハーブブレンドを楽しむ人が、自分のレシピを再現可能な形で記録し、経験を蓄積できることを目的としたサービスです。

# 2. このアイデアはどこから生まれたか

## 2-1. きっかけとなった体験・感情

ハーブブレンドを自作する中で、
「美味しかった配合を忘れる」「苦手だったブレンドを再度作ってしまう」
という失敗を繰り返していました。
スマートフォンのメモやノートでは、
ハーブの割合・味の印象・体調の変化などを体系的に整理できず、
「あの一杯」を再現できないもどかしさを感じたことが出発点です。
また、自分の好みに合うレシピを探そうとしても、
既存のネット検索では理想のブレンドに辿り着けないという課題にも直面しました。

## 2-2. なぜそれが「気になった」のか

ハーブティーは、味の好みや気分に合わせて微調整する楽しさがある一方で、
その試行錯誤が「データ」として蓄積されない限り、個人の経験を次に活かすことができません。
現状、ブログやInstagramなどでは魅力的なブレンドが数多く発信されていますが、
情報が埋もれてしまいやすく、自分に合ったレシピを検索・再現することが極めて困難です。

また、ブレンド初心者にとっては、どの単品ハーブを揃えれば良いかの判断が難しく、
購入のハードルが高いという現状もあります。
ハーブブレンドは本来、経験の積み重ねが重要であるにも関わらず、有益な知見が埋もれてしまい、
初心者が一歩踏み出しにくい環境にあることに強い違和感を抱きました。

# 3. 課題の整理

## 3-1. 表に見えている困りごと

記録の散乱： 配合、味、感想を体系的に整理・記録するのが困難。

検索の困難さ： 自分の嗜好や手持ちのハーブに合致するレシピが見つからない。

## 3-2. 本当に解決したい課題

- 「ハーブブレンドの試行錯誤を、再現可能な形で資産化できない」という問題の解消。

- 直感的な記録と振り返りを通じ、自分の「好き」という感覚を確かなレシピ（資産）として確立できる体験の提供

- 個人のブレンドレシピと風味記録をデータ化し、経験や知識を他者と共有・活用できるプラットフォームの実現。

# 4. 想定ユーザーについて

## 4-1 想定しているユーザー

①自宅に数種類のハーブをストックし、ブレンドを施行している人

②ハーブティーのブレンドに挑戦したい初心者

生活・状況設定：

- 生活・状況：

  自宅でハーブティーをブレンドする習慣があり、いろんなパターンで試している。
  しかし、うまくできた時の分量を記録しても後からレシピメモを探すのが困難。

- 理由：

  タグ機能を使った使用ハーブ検索機能や、風味検索によってすぐに目当てのブレンドが見つけられる。

環境設定：

- 環境：

  ブレンドしながらメモを取ってもページを見失ったり、メモを無くしてしまう。

- 理由：

  データで管理し、風味タグや評価を統一できるため後からブレンド内容と感想を見返しやすい。

## 4-2 自分とユーザーの距離

自分自身がハーブブレンドを試す中で、レシピや味の記録方法や、
手元にあるハーブの種類に合わせたブレンドをレシピメモから抽出することに
不便さを感じた経験があったため。

## 4-3 実際に使われる可能性について

最初に使ってくれそうな人：

- ハーブティーのブレンドを趣味としている人。
- 自分の好みに合う風味のハーブティーを探している人。

どんな場面で思い出すか：

- 新しいハーブブレンドを作ったとき
- 過去のブレンドレシピを見返すとき
- 手元にあるハーブでブレンドを考えるとき
- 自分と好みの近いユーザーのブレンドをチェックするとき

今使っている代替手段：

- スマートフォンのメモ
- 紙のノート
- Chat GPT

  それを置き換えてまで使う理由：

  ハーブブレンドの

- 配合
- 風味
- ブレンドを作ってみた感想

  などを1つの場所で整理できるため。

## 4-4 ユーザーが導入して利用するまで

### サービスを知るきっかけ

- ハーブティーのブレンドレシピを検索するとき。

### 最初の一歩

- ブレンドレシピを検索

### 利用を続ける理由

- 他の人のブレンドを参考にできる
- 自分の好きなブレンド履歴が蓄積される
- 手持ちのハーブを使ったブレンドを検索できる

写真や文章での表現には優れるが、「好みに合うブレンドを探す」「手持ちのハーブから作れるブレンドを検索する」といったデータベース的な活用が困難。

## 5-2 それでも自分が作りたい理由

既存のレシピサービスは主に料理レシピを対象としており、

- ハーブブレンドの配合
- 味の印象
- 体調への効果

  などを体系的に整理することには向いていないと感じました。
  ハーブブレンドに特化した記録と共有の仕組みを持つサービスが少ないことが自分で作りたいと思った理由です。

## 5-3 差別化

本アプリは、単なるレシピ記録を超え、**「個人の嗜好に徹底的に寄り添う」**体験を提供します。

ポジティブな味ベース検索：
効能だけでなく「甘み」「スパイシー」「フローラル」といった味の好みからレシピに辿り着ける設計。

ブレンド検索時の除外フィルタリング：
苦手なハーブや禁忌（カフェイン、特定成分）をあらかじめ設定することで、ブレンドを検索時に「このブレンドは苦手なハーブが入ってるから飲めないな」と諦めることがなくなります。

嗜好の資産化：
「この配合は好き」「これはミントが強すぎた」という試行錯誤を蓄積し、自分だけの「至高の一杯」を実現させ、またそれを再現可能にします。

「購買」ではなく「研究（記録）」への特化:
既存のサイトの多くは「売るためのレシピ」や「完成された商品」が主役です。対して「Herb Recipe Lab」は、「手元の素材でどう美味しくするか」という試行錯誤の過程そのものを記録することができます。

# 6. このサービスで提供したい価値

## 6-1 ユーザーの変化

- 行動の変化：ブレンドレシピを継続的に記録するようになる。
- 気持ちの変化：自分のブレンド経験が知識として蓄積される安心感が生まれる。
- 考え方の変化：ハーブブレンドを試行錯誤を楽しむ「実験」のように考えられるようになる。

## 6-2 価値を一文で表す

「ハーブティーブレンドの配合・味・効果を記録し、レシピを蓄積・共有できるアプリ」

# 7. このアプリで実現すること

## MVPで作る機能

- ユーザー認証機能

  メールアドレスによる新規登録、ログイン、ログアウト機能。

- ハーブ管理機能（マスタ・自由入力）

  50音順に整理されたハーブマスタから選択。

- マスタにない材料は「その他」として自由記述で追加可能。

- 二段階保存によるブレンド研究記録

  STEP 1（淹れる前）: 日付、タイトル、ハーブ配合、200mlあたりの分量（g/小さじ単位統一切り替え）を記録。
  STEP 2（飲用後）: 実際の味や香りの感想を、レシピを振り返りながら追記。

- レシピ共有機能

  投稿されたレシピの一覧表示、詳細表示。

## 本リリースで作る機能

- 5段階評価 ＆ 8要素の風味チャート
  「甘味・苦味・渋味・清涼感・スパイシー・清涼感・フルーティー・酸味・華やかさ」の8つの指標で味を客観的に可視化。
  自己分析や「好みの近いユーザー」とのマッチングの基礎データとなります。

- ハーブ辞典（Turbo Stream実装）

  レシピ入力画面を離れることなく、非同期でハーブの画像・風味・効能・禁忌を確認できる機能。
  50音順、風味ジャンル、効能ジャンルでの並び替え・絞り込みに対応。

- 多角的なレシピ検索機能

  使用ハーブ、期待する効果、好みの風味タグを組み合わせたクロス検索。

- コミュニティ・ストック機能

  ユーザー間のフォロー機能、ブックマーク、他ユーザーのブレンドレシピの評価機能。

- 外部API（Gemini API）連携によるブレンド提案

  手持ちのハーブから、相性の良い組み合わせや期待できる効果をAIがアドバイス。

- 市販ハーブティー・飲用ログ機能

  自作レシピだけでなく、既製品の飲用感も同様のチャートで記録可能に。

# 8. このアプリの懸念点とその対策

## 懸念点

- 何が問題になりそうか：ハーブブレンドをするユーザー数が限定的である可能性。

- なぜそう思うか：ハーブブレンドは一般的な料理や飲み物と比べ、趣味人口が比較的少ない分野であるため。

- 対策：ハーブブレンドだけでなく、市販ハーブティーの飲用記録を投稿できるようにすることでハーブティー初心者でも飲んでみたいと思ってもらえるようにする。

## ユーザーが使い続ける理由

ブレンドレシピが増えることで、自分だけのハーブデータベースが形成される。
また、好みの近いユーザーのブレンドを参考にできる。

## データが溜まるとできること

- 人気ブレンドの分析
- 好みの味や求める効果の傾向分析

## 今考えている発展案

### 機能追加の方向性

- AIによるブレンド提案 → そのまま記録に反映できる
- 苦手なハーブ、体調・体質的に禁忌なハーブを除外検索する機能

### UI・体験の改善

- 味のマッピング表示
- 香りのチャート表示

### 機能以外の工夫

- ハーブの基本情報を表示する辞典機能
- ハーブティー以外のハーブの用途を記載したコラム投稿機能

# 10. 技術スタック

## 10-1 使用予定の技術

- 開発言語：Ruby3.4.x
- フレームワーク：Ruby on Rails7.2.x
- データベース：PostgreSQL
- デプロイ先：Render
- 開発環境：Docker
- 認証：devise
- 外部API連携：Gemini API
- CSS／UIフレームワーク：Tailwind CSS
- その他ライブラリ／Gem
  - RSpec
  - Devise（ユーザー認証）
  - TailwindCSS（UI）
  - Ransack（検索機能）
  - kaminari（ページネーション）
  - acts-as-taggable-on（自由タグ）

## 10-2 技術選定の理由

### なぜこの技術を使うのか

- Ruby on Railsは開発速度が速く、個人開発のWebサービスに適しているため。

### 今回チャレンジしたい点

- レシピ保存→感想保存、投稿の２段階設計
- 検索機能の実装
- タグ機能の設計
- レシピ共有機能と公開・非公開設定

### 不安な点

- アプリ使用方法の解説
- ブレンドレシピ検索のためのデータ構造や検索設計
- 味の好み、求める効果をわかりやすく表示するチャート設計
